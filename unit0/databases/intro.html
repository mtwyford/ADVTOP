What is a Database?

  • Database: A structured collection of data that is organized and stored electronically
  • Purpose: To efficiently store, retrieve, and manage large amounts of information
  • Examples: Student records, online shopping catalogs, social media posts, banking systems
  • Benefits: Data organization, quick access, data integrity, concurrent access by multiple users
Think of it like: A digital filing cabinet where information is organized in folders and can be quickly found and updated!

Database vs Traditional File Storage

File System Database System

Data redundancy and inconsistency Reduced redundancy, data consistency

Relational Databases

  • Relational Database: Data is stored in tables (relations) with rows and columns
  • Tables: Each table represents an entity (students, courses, orders)
  • Rows (Records): Each row contains data about one instance
  • Columns (Fields): Each column represents an attribute or property
  • Relationships: Tables can be connected through common fields

Database Management Systems (DBMS)

  • Popular DBMS:

  • Why MySQL? Free, reliable, widely used, great for learning

Database Normalization

  • Normalization: Process of organizing data to reduce redundancy and improve data integrity
  • Goals:
    • Eliminate redundant data
    • Ensure data consistency
    • Reduce storage space
    • Make updates easier and safer
  • Normal Forms: Rules that define levels of organization
  • We'll cover: 1NF, 2NF, and 3NF

First Normal Form (1NF)

Rule: Each cell must contain only atomic (single) values

  • No repeating groups or arrays in a single cell
  • Each column should contain only one type of data
  • Each record should be unique

❌ NOT in 1NF:

StudentIDNameSubjects
1JohnMath, Science, English

✅ Correct 1NF:

StudentIDNameSubject
1JohnMath
1JohnScience
1JohnEnglish

Second Normal Form (2NF)

Rule: Must be in 1NF + No partial dependencies on composite keys

  • All non-key columns must depend on the entire primary key
  • Eliminates redundancy caused by partial dependencies
  • Usually involves splitting tables

❌ NOT in 2NF:

StudentIDCourseIDGradeStudentNameCourseName
1101AJohnMath

Problem: StudentName depends only on StudentID, not the full key (StudentID + CourseID)

✅ Correct 2NF:

Students
StudentIDStudentName
1John
Enrollments
StudentIDCourseIDGrade
1101A

Third Normal Form (3NF)

Rule: Must be in 2NF + No transitive dependencies

  • Non-key columns should not depend on other non-key columns
  • Eliminates transitive dependencies
  • Further reduces redundancy

❌ NOT in 3NF:

StudentIDNameDepartmentIDDepartmentNameDepartmentHead
1JohnCSComputer ScienceDr. Smith

Problem: DepartmentName and DepartmentHead depend on DepartmentID, not StudentID

✅ Correct 3NF:

Introduction to MySQL

  • MySQL: Popular open-source relational database management system
  • SQL: Structured Query Language - used to communicate with databases
  • Key Features:
    • Fast and reliable
    • Easy to use
    • Runs on many operating systems
    • Free and open source
    • Used by major websites (Facebook, YouTube, Netflix)
  • SQL Command Types: DDL (Data Definition), DML (Data Manipulation), DQL (Data Query)

Creating Tables with MySQL

CREATE TABLE Syntax:

-- Create a database first CREATE DATABASE school_db; USE school_db; -- Create students table CREATE TABLE students ( student_id INT AUTO_INCREMENT PRIMARY KEY, first_name VARCHAR(50) NOT NULL, last_name VARCHAR(50) NOT NULL, email VARCHAR(100) UNIQUE, grade_level INT, enrollment_date DATE ); -- Create courses table CREATE TABLE courses ( course_id INT AUTO_INCREMENT PRIMARY KEY, course_name VARCHAR(100) NOT NULL, course_code VARCHAR(10) UNIQUE, credits INT DEFAULT 3, instructor VARCHAR(100) );

Altering Tables

ALTER TABLE - Modify Existing Tables:

-- Add a new column ALTER TABLE students ADD COLUMN phone_number VARCHAR(15); -- Modify existing column ALTER TABLE students MODIFY COLUMN email VARCHAR(150); -- Drop a column ALTER TABLE students DROP COLUMN phone_number; -- Rename a column ALTER TABLE students CHANGE COLUMN grade_level grade INT; -- Add a foreign key constraint ALTER TABLE enrollments ADD CONSTRAINT fk_student FOREIGN KEY (student_id) REFERENCES students(student_id);

Basic SELECT Statements

Retrieving Data from Tables:

-- Select all columns from students table SELECT * FROM students; -- Select specific columns SELECT first_name, last_name, email FROM students; -- Select with column aliases SELECT first_name AS 'First Name', last_name AS 'Last Name', grade AS 'Grade Level' FROM students; -- Select distinct values SELECT DISTINCT grade FROM students; -- Limit results SELECT * FROM students LIMIT 5;

SELECT with WHERE Clause

Filtering Data:

-- Basic WHERE conditions SELECT * FROM students WHERE grade = 11; -- Multiple conditions with AND SELECT * FROM students WHERE grade = 12 AND first_name = 'Sarah'; -- Multiple conditions with OR SELECT * FROM students WHERE grade = 11 OR grade = 12; -- Using IN for multiple values SELECT * FROM students WHERE grade IN (9, 10, 11); -- Pattern matching with LIKE SELECT * FROM students WHERE last_name LIKE 'Sm%'; -- Range with BETWEEN SELECT * FROM students WHERE grade BETWEEN 10 AND 12;

UPDATE and JOIN Statements

UPDATE - Modifying Data:

-- Update a single record UPDATE students SET email = 'john.doe@email.com' WHERE student_id = 1; -- Update multiple columns UPDATE students SET grade = 12, email = 'sarah.johnson@email.com' WHERE student_id = 2;

JOIN - Combining Tables:

-- INNER JOIN - students with their enrollments SELECT s.first_name, s.last_name, c.course_name, e.grade FROM students s INNER JOIN enrollments e ON s.student_id = e.student_id INNER JOIN courses c ON e.course_id = c.course_id; -- LEFT JOIN - all students, even if not enrolled SELECT s.first_name, s.last_name, c.course_name FROM students s LEFT JOIN enrollments e ON s.student_id = e.student_id LEFT JOIN courses c ON e.course_id = c.course_id;
